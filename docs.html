The Deployment Bible

This manual outlines the technical implementation of the Ranasinghe Infra AWS Kit. It is designed for DevOps Engineers and Architects looking to deploy a SOC2-compliant production environment in minutes.

Prerequisites

Tooling

Terraform v1.5+, AWS CLI v2, and kubectl must be installed in your local shell or CI/CD runner.

Permissions

IAM User with AdministratorAccess or a cross-account role with trust relationships established.

01. Remote State Locking

To prevent state corruption during concurrent team deployments, we utilize an S3 Backend with DynamoDB for state locking.

terraform {
  backend "s3" {
    bucket         = "ranasinghe-infra-terraform-state"
    key            = "prod/infrastructure.tfstate"
    region         = "us-east-1"
    dynamodb_table = "terraform-lock-table"
    encrypt        = true
  }
}

02. VPC & Isolated Networking

Our module.vpc creates a triple-tier subnet architecture across 3 Availability Zones (AZs).

  • A

    Public Layer

    Hosts Internet Gateways and ALBs. No application servers reside here.

  • B

    Private Layer

    Hosts EC2/EKS nodes. Outbound traffic via NAT Gateways only.

  • C

    Isolated Layer

    Hosts RDS and ElastiCache. Zero external route path.

03. Security & IAM Hardening

Every asset is injected with a Default_Tags variable for cost tracking and security auditing.

module "security_group" {
  source = "./modules/security"
  vpc_id = module.vpc.id
  ingress_rules = [
    { port = 443, cidr = ["0.0.0.0/0"], desc = "Allow HTTPS" }
  ]
}

04. EKS Orchestration

The final step deploys a managed Kubernetes cluster with Amazon Linux 2 optimized nodes and AWS Load Balancer Controller integration.
